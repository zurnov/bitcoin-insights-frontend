2. Audit Categories

2.1 Technical SEO

Score: 35/100

What Works

  • HTTPS on main domain via Cloudflare
  • HTTP/2 and HTTP/3 (alt-svc h3) enabled
  • Gzip compression (460KB main.js → 142KB)
  • Cloudflare CDN with fast TTFB (~92ms from SOF edge)
  • Critters critical CSS inlining active
  • html lang='en' set correctly

Findings

robots.txt Returns HTML (Angular SPA Intercept) Critical

GET /robots.txt returns text/html with Angular SPA shell (HTTP 200). Googlebot cannot parse crawl directives. All well-known file paths (sitemap.xml, llms.txt, security.txt) return the same SPA shell.

Recommendation: Create /src/assets/robots.txt as a static file. Configure Nginx: location = /robots.txt { root /var/www/explore21/assets; try_files /robots.txt =404; }

sitemap.xml Returns HTML Critical

GET /sitemap.xml returns text/html (200 OK). No XML sitemap exists. Google cannot discover or inventory site pages.

Recommendation: Create static sitemap.xml listing homepage and key tool pages. Serve like robots.txt. Submit to Google Search Console.

HTTP Not Redirected to HTTPS Critical

http://explore21.com returns 200 OK over plain HTTP. No redirect to HTTPS. Users on HTTP are exposed to MITM attacks.

Recommendation: Enable Cloudflare 'Always Use HTTPS' in SSL/TLS settings.

www Redirects to HTTP Instead of HTTPS Critical

https://www.explore21.com issues 301 redirect to http://explore21.com/ — downgrading from HTTPS to HTTP. This is a security vulnerability.

Recommendation: Add Cloudflare Redirect Rule: www.explore21.com → https://explore21.com${uri} (301)

No Security Headers High

Missing: Strict-Transport-Security, Content-Security-Policy, X-Frame-Options, X-Content-Type-Options, Referrer-Policy, Permissions-Policy. Site is vulnerable to clickjacking, MIME sniffing, and protocol downgrade attacks.

Recommendation: Add via Cloudflare Transform Rules: X-Frame-Options: SAMEORIGIN, X-Content-Type-Options: nosniff, Referrer-Policy: strict-origin-when-cross-origin, HSTS: max-age=31536000

Angular CSR-Only — No Server-Side Rendering High

Pure client-side Angular SPA. HTML shell contains only <app-root></app-root>. All meta tags are static — every page shares the same title/description. Googlebot must execute JavaScript to see any content.

Recommendation: Implement Angular Universal SSR (ng add @nguniversal/express-engine) or Angular 17+ built-in SSR. Minimum: implement Angular Meta service + Router-based meta tag updates per route.

No Canonical Tag High

No <link rel='canonical'> found. Same content accessible via http://, https://, and potentially with/without trailing slash.

Recommendation: Add canonical tag to index.html. Update dynamically per route via Angular's Meta service.

2.2 Content Quality

Score: 28/100

What Works

  • Meta keywords include relevant terms
  • Tool appears functional (Bitcoin explorer)

Findings

Zero Indexable Content — JS-Rendered SPA Critical

HTML shell contains no visible text content. <app-root></app-root> is empty without JS execution. Crawlers without JS execution see no content whatsoever.

Recommendation: Implement SSR to server-render at minimum the homepage content.

E-E-A-T Score: 18/100 Critical

No Experience or Expertise signals. No About page, no team info, no data source citations, no contact info. Trustworthiness extremely low for a financial tool (crypto explorer).

Recommendation: Create About, Privacy, Terms, Contact pages. Add data source attribution. Consider adding team/founder information.

Brand/Domain Mismatch High

Product is called 'BTC Insights' but domain is explore21.com. No brand alignment. Users won't remember the domain. AI systems will be confused about citations.

Recommendation: Either rebrand to align with domain (explore21.com) or acquire btcinsights.com or btcinsights.io.

Missing Essential Pages High

No /about, /privacy, /terms, /contact, /faq pages with real content. All return the SPA shell. These are E-E-A-T signals and trust factors, especially for a crypto tool.

Recommendation: Create static content pages with 300+ words each. Start with Privacy Policy and About page.

No Content Hub Medium

Tool-only site with no educational content. Competitors (mempool.space, blockchain.com) have rich blog/resource sections capturing long-tail traffic.

Recommendation: Create /learn or /blog with Bitcoin explorer tutorials and concept explanations.

2.3 On-Page SEO

Score: 32/100

What Works

  • Meta description 112 chars (acceptable length)
  • og:title and og:description present
  • og:type: website set
  • charset and lang attributes correct

Findings

Weak Title Tag — No Keywords Critical

Title is 'BTC Insights' (11 chars). No target keywords. Does not describe page purpose to users or search engines.

Recommendation: Change to: 'Bitcoin Blockchain Explorer – Address & Transaction Lookup | BTC Insights'

Missing og:image High

No Open Graph image. Shared links on Discord/Telegram/X show no preview. Critical for crypto community engagement.

Recommendation: Create 1200x630px branded image. Add og:image, og:image:width, og:image:height meta tags.

Missing Twitter Card High

No twitter:card meta tag. X/Twitter shows no rich preview for shared links.

Recommendation: Add: <meta name='twitter:card' content='summary_large_image'> and related twitter:* tags.

Missing og:url and og:site_name Medium

Open Graph URL and site name not specified.

Recommendation: Add: <meta property='og:url' content='https://explore21.com/'> and <meta property='og:site_name' content='BTC Insights'>

Viewport Blocks Zoom — Accessibility Violation Medium

user-scalable=no and maximum-scale=1 prevent zooming. Violates WCAG 1.4.4 (Resize Text, Level AA).

Recommendation: Change to: <meta name='viewport' content='width=device-width,initial-scale=1'>

2.5 Performance (CWV)

Score: 45/100

What Works

  • Fast TTFB: ~92ms (Cloudflare CDN)
  • HTTP/2 + HTTP/3
  • Gzip compression enabled
  • Critical CSS inlined (Critters)
  • Non-critical CSS deferred (media=print onload trick)

Findings

FontAwesome Synchronous Load — Render Blocking Critical

<script src='kit.fontawesome.com/...'> with no async or defer. Blocks rendering by 300-500ms.

Recommendation: Add defer attribute: <script defer src='https://kit.fontawesome.com/8a3442c427.js' crossorigin='anonymous'>

Google Fonts via @import — Worst Loading Pattern High

@import url(fonts.googleapis.com) in inline CSS blocks CSS parsing, delaying FCP by 200-400ms.

Recommendation: Replace with: <link rel='preconnect' href='https://fonts.googleapis.com'> + <link rel='stylesheet' href='...'> in <head>.

Angular CSR — JS-Gated LCP (3-7s on mobile) High

Largest Contentful Paint blocked until 460KB JS loads and executes. Estimated LCP: 3-7s on typical mobile connection.

Recommendation: Implement Angular Universal SSR to server-render first paint.

Short Cache TTL on Hashed Static Assets Medium

max-age=63,675s (~17h) on hash-named JS/CSS files. Files with content-hash are safe to cache for 1 year.

Recommendation: Set Cache-Control: public, max-age=31536000, immutable for all *.hash.js and *.hash.css files.
